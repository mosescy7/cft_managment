Going live on cft.rw — full checklist

Getting the site live at cft.rw

From today's state (GitHub Pages at mosescy7.github.io/cft_managment/) to a working cft.rw

Phase 1 — Register the domain

  1. Go to RICTA's accredited registrar list: https://www.ricta.org.rw/accredited-registrars/
  2. Pick one and search availability for cft.rw.
  3. If available, register it — no residency/citizenship requirement, registration is usually done within minutes of payment. Register for at least 1 year (max 5 years available).
  4. Keep the registrar login details — you'll need to log back in for Phase 3 (DNS).

(If cft.rw turns out to be taken, fall back to cftapartments.rw or cftapartments.com — everything below works identically for any domain, just substitute it in.)

Phase 2 — Tell GitHub Pages about the domain

You've already started this: repo → Settings → Pages → Custom domain.

  1. Type cft.rw in that field and click Save.
    • This is safe to do even before DNS is configured — it just creates a CNAME file in the repo root telling GitHub "serve this site for cft.rw."
    • GitHub will likely show a "DNS check unsuccessful" warning right after saving — expected at this stage, ignore it until Phase 3 is done.

Phase 3 — Point the domain's DNS at GitHub

Log into the registrar from Phase 1, find its DNS management panel, and add these records:

Type Host / Name Value
A @ (or blank/root) 185.199.108.153
A @ (or blank/root) 185.199.109.153
A @ (or blank/root) 185.199.110.153
A @ (or blank/root) 185.199.111.153
CNAME www mosescy7.github.io
  • The four A records make cft.rw (no "www") work.
  • The CNAME makes www.cft.rw redirect to the same site.
  • Remove/replace any existing "parking page" A or CNAME records the registrar auto-added when you registered — they'll conflict with these.

Phase 4 — Wait, verify, enable HTTPS

  1. DNS changes typically take a few minutes to a few hours to propagate; occasionally up to 24-48h.
  2. Once propagated, go back to Settings → Pages in the repo — the "DNS check unsuccessful" warning should clear on its own.
  3. Tick "Enforce HTTPS" once it's selectable (GitHub auto-issues a free SSL certificate at this point — this can take up to a few hours after DNS clears).
  4. Test it: visit https://cft.rw and https://www.cft.rw — both should load the site securely.

Phase 5 — Cleanup after it's live

Nothing in the site's code needs to change — all internal links are relative, so the domain swap is transparent to the HTML/CSS. Just a few loose ends elsewhere:

  • Google Business Profile — update the website field from the github.io link to https://cft.rw.
  • Google Ads campaign (if you went ahead with one) — update the final URL to https://cft.rw.
  • README.md — line 44 currently says https://mosescy7.github.io/cft_managment/index.html; update it to the new domain (cosmetic, low priority — happy to do this one whenever you want).
  • Email — if you set up Google Workspace/Zoho on this same domain, that's independent of these DNS records (different record types: MX, not A/CNAME) — don't let one setup overwrite the other's DNS entries.
